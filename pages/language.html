Language and Translations.

When you create a new page, you must define the languages available for translation as well as the scope of the translations. The jekyll-language-plugin also exposes a variable that will alter the url of the page to include the language.

{% highlight html %} {% raw %} --- layout: default title: Workopolis | Testing Languages with Collections and Data permalink: /:language/langtest/ languages: - en - fr subset: allsolutions --- {% endraw %} {% endhighlight %}

We use jekyll-language-plugin for for page translations. General translation are stored in _date/lang.yml. Each page should have its own section in the lang.yml file.

{% highlight html %} {% raw %} en: allsolutions: slug: allsolutions heroTitle: Take the stress out of hiring heroText: Hiring shouldn’t be complicated. That’s why each of our solutions was built with you in mind. fr: allsolutions: slug: allsolutionsfr heroTitle: L’embauche sans stress. heroText: L’embauche ne devrait pas être un exercice compliqué. Voilà pourquoi chacune de nos solutions a été conçue en fonction de vos besoins. {% endraw %} {% endhighlight %}

To reference a translation in your page you can reference using the t Liquid tag followed by the name of the resource.
