Jekyll - Workopolis Static Pages

Using a static-site generator to build landing pages for Workopolis.

Page Creation and Routes

Static pages can be created in the pages folder. At the top of each HTML page you can define things like page layout, title and url. To define the pages URL just add a permalink to the Front Matter of the HTML file:

{% highlight html %} --- layout: default title: Workopolis | Post A Job permalink: /post-a-job/ --- {% endhighlight %}

The Job Posting page will now be available at here

Working with Collections

Collections allow us to create structured data that can be referenced anywhere on the site. New Collections are defined in _config.yml and the collection data is stored in the folder matching the name of the collection.

{% highlight html %} name: Worko markdown: kramdown highlighter: rouge exclude: ["node_modules", "gulpfile.js", "package.json"] port: 4000 collections: products: {% endhighlight %}

Each record in the collection contains Front Matter and Markdown. Information defined in Front Matter can be used to output specific data from the collection

This demo has a Collection called _products that contains information on a few Workopolis B2B products. To output this information we can use Liquid templats to loop through the collection:

{% highlight html %} {% raw %} {% for product in site.products %}

{{product.title}}

{{product.price}}

{{product.short-description}}

Buy Now For {{product.price}}
{% endfor %} {% endraw %} {% endhighlight %}

This will generate markup for each entry in the collection:

{% for product in site.products %}

{{product.title}}

{{product.price}}

{{product.short-description}}

Buy Now For {{product.price}}
{% endfor %}

Working with Data

Jekyll also alllows you to define custome data that can be accessed through Liquid templates. Custome data can be created in YAML, JSON or CSV. A collection of data containing Workopolis' social links could be looped through anywhere on the site without manually entering the same data repeatedly.

social.yml
{% highlight html %} - name: twitter url: https://twitter.com/Workopolis - name: facebook url: https://facebook.com/Workopolis - name: instagram url: https://instagram.com/workopolis {% endhighlight %}
Liquid Template
{% highlight html %} {% raw %}
    {% for social in site.data.social %}
  • {{ social.name }}
    • {% endfor %}
{% endraw %} {% endhighlight %}
Markup
    {% for social in site.data.social %}
  • {{ social.name }}
    • {% endfor %}
