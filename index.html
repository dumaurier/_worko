Worko Jekyll.

This guide will help you get set up to run Jekyll on your local machine.

Getting set up.

Requirements:

  1. Ruby

    Windows: Download and install from here: http://rubyinstaller.org/

    Mac: Guess what? Ruby is installed by default. Fun!

  2. Bundler

    We use Bundler to manage installion of Ruby Gems required by Jekyll. To install Bundler open a Terminal or Command window and run: 

    $ gem install bundler --source http://rubygems.org
    Windows will throw random SSL errors if you don't force Gems to download over http which is why we have to change the source download. If you're using a Mac you can omit the --source option.

  3. Node.js

    Node isn't required to run Jekyll itself but we use it for our development environment and asset pipeline. Download Node.js here: https://nodejs.org/en/

  4. Gulp

    Gulp is fun to say. It's also helps us develop new pages in Jekyll. To install Gulp, open a Terminal or Command window and run 

    $ npm install --global gulp-cli

  5. Git

    Create a folder called Jekyll. Open a Terminal or Command window inside your newly created folder and run 

    $ git init
    . Then clone the Worko Jekyll repo: 
    $ git clone https://github.com/dumaurier/_worko.git

  6. Jekyll

    Change your directory to the new _worko repo: 

    $ cd _worko
    And run: 
    $ bundle install
    This will install Jekyll and any other required Gems.

  7. NPM Modules

    Gulp has several dependcies to run our development environment. To install them run: 

    $ npm install

  8. Build the site.

    That's it. From your Terminal or Command window run: 

    $ gulp
    This will build the Jekyll site, start the asset pipeline and set up BrowserSync for live reload.
